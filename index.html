Rubriker

Wigellkoncernen använder Montserrat för alla rubriker för att ge ett modernt och stabilt intryck. Kopiera den gemensamma css-koden samt css för de rubriker du använder.

CSS
h1, h2, h3, h4 {
    color: #2D5E4A;
    font-family: 'Montserrat', sans-serif;
    font-weight: 700;
    margin-bottom: 1rem;
    line-height: 1.2;
}

H1: Huvudrubrik

HTML
<h1>Huvudrubrik</h1>
CSS
h1 { 
    font-size: 2.5rem; 
}

H2: Sektionsrubrik

HTML
<h2>Sektionsrubrik</h2>
CSS
h2 {
    font-size: 2rem; 
    border-bottom: 2px solid #AAB3A3; 
    padding-bottom: 5px; 
}

H3: 1. Underkategori

HTML
<h3>Underkategori</h3>
CSS
h3 {
    font-size: 1.5rem; 
}

H4: Liten rubrik / Etikett

HTML
<h4>Liten rubrik / Etikett</h4>
CSS
h4 { 
    font-size: 1.2rem; 
    text-transform: uppercase; 
    letter-spacing: 1px; 
}

Brödtext

För längre texter och stycken använder vi en väl tilltagen radhöjd för att säkerställa god läsbarhet på alla skärmar.

Detta är ett exempel på hur ett vanligt textstycke ser ut. Wigellkoncernen strävar alltid efter tydlig kommunikation och professionalism. Genom att använda rätt radavstånd blir texten lätt att skanna av och behaglig att läsa även under längre sessioner.

Tips: Använd korta stycken för att göra innehållet mer lättillgängligt för användaren.

HTML
<p>Text <strong>text</strong> text.</p>
CSS
p {
    font-family: 'Montserrat', sans-serif;
    font-size: 1rem;
    line-height: 1.6;
    color: #333333;
    margin-bottom: 1.5rem;
}
strong {
    font-weight: 700;
    color: #2D5E4A;
}

em {
    font-style: italic;
}

Knappar

Vi använder fyra huvudtyper av knappar för att guida användaren rätt. Alla knappar anväder samma grund-css, kopiera nedan.

CSS
.btn {
    padding: 12px 24px;
    border: none;
    border-radius: 4px;
    font-family: 'Montserrat', sans-serif;
    font-weight: 600;
    cursor: pointer;
    transition: background-color 0.3s, transform 0.1s;
    display: inline-block;
}

.btn:active {
    transform: scale(0.98);
}

Används för generella handlingar som inte kräver extra uppmärksamhet.

HTML

    color: white;
}
.btn-standard:hover {

Laddningsindikator

Används för att visa att systemet bearbetar data. Vi använder en cirkulär animation för att indikera väntetid.

Laddar innehåll...
Bearbetar ändringar...
HTML
<div class='loader-container'>
    <span class='loader'></span>
    <span class='loader-text'>Laddar innehåll...</span>
</div>
CSS
.loader {
    width: 48px;
    height: 48px;
    border: 5px solid #AAB3A3;
    border-bottom-color: #2D5E4A;
    border-radius: 50%;
    animation: rotation 1s linear infinite;
}

@keyframes rotation {
    0% { transform: rotate(0deg); }
    100% { transform: rotate(360deg); }
}

    <label>Namn</label>

  </div>
  <button class='btn btn-standard'>Skicka</button>

    max-width: 500px;
    background: #ffffff;
    padding: 20px;
    border-radius: 8px;
}

.form-group {
    margin-bottom: 1.5rem;
    text-align: left;
}

    display: block;
    font-family: 'Montserrat', sans-serif;
    font-weight: 600;
    margin-bottom: 0.5rem;
    color: #2D5E4A;
}

.form-group input, 
.form-group textarea,

    width: 100%;

    border-radius: 4px;

.form-group input:focus, 
.form-group textarea:focus {

    border-color: #2D5E4A;
    box-shadow: 0 0 5px rgba(45, 94, 74, 0.2);

Färgschema

Vår palett är inspirerad av naturen, vilket speglar Wigells bredd och stabilitet. För vanlig brödtext används svart färg för tydlighet, i rubriker används Wigell-grön.
För tydlighet används gärna root-variabler i CSS.

Wigell-grön #2D5E4A
Bakgrund i main #D5E0CC
Sidebar, funktionsknapp, neutral #AAB3A3
Bakgrund, neutral #E6E6E6
Negativa knappar #97351C
Bakgrund, negativ #FDF2F2
Positiva knappar #4CAF50
Bakgrund, positiv #e8f5e9
CSS
:root {
    --wigell-color: #2D5E4A;
    --background-main-color: #D5E0CC;
    --background-sidebar-color: #AAB3A3;
    --background-neutral-color: #E6E6E6;
    --negative-color: #97351C;
    --background-negative-color: #FDF2F2;
    --positive-color: #4CAF50;
    --background-positive-color: #E8F5E9;
}
    /*Ex. användning av variabel*/
    background-color: var(--background-main-color);

Kod och namnsättning

I koden använder vi ' istället för " för ett modernare intryck. Som namnstandard, använd beskrivande namn, all kod och kommentarer skrivs på engelska. Id och class-namn används ett beskrivande namn så som 'btn-positive', btn för att förklara att det handlar om en knapp och -positive som förklaring vad det hör till.
I JavaScript används camelCase, ex toggleMenu(), i CSS och HTML används kebab-case, ex btn-standard, används klasser används PascalCase, ex UserInterface.
För filnamn används små bokstäver.

Standard för tab är fyra space.
